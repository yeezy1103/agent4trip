# 前端视觉与排版全面改造计划

## Summary
- 目标：在不改变现有业务逻辑、接口调用顺序、核心展示内容的前提下，对前端 `Home` 与 `Result` 两页做一次“像完全不同产品”的全站级视觉重构。
- 风格方向：统一为“苹果玻璃感”，但动效节奏保持克制顺滑，不做过度炫技。
- 范围：覆盖全局壳层、设计令牌、首页首屏与表单、结果页信息架构、地图、预算、概览、每日行程、景点卡的视觉层次与排版节奏。
- 约束：保留业务逻辑与现有数据内容；允许重构页面布局与 DOM 分组；全站统一玻璃语言；结果页优先强化整体层次而非单一模块。

## Current State Analysis

### 已确认的当前结构
- 应用壳层在 `frontend/src/App.vue`，当前仍是基础 `a-layout` + 深色头部 + 普通内容区。
- 路由入口在 `frontend/src/main.ts`，当前只挂载 `Home` 和 `Result` 两个页面。
- 全局视觉基线在 `frontend/src/design-system.css`，目前是“现代化 token + Ant Design 覆盖”的一版中性设计系统。
- 首页在 `frontend/src/views/Home.vue`，当前是三段式表单页：标题区、三组表单区块、提交按钮与进度条。
- 结果页在 `frontend/src/views/Result.vue`，当前包含顶部操作区、侧边导航、概览、预算、地图、每日折叠行程、景点卡、酒店推荐、餐饮安排。

### 当前视觉问题
- `App.vue` 的整体壳层仍比较通用，缺少品牌化的第一印象，也没有形成苹果式空间感。
- `design-system.css` 的 token 偏中性 SaaS 风格，和“全站统一玻璃感”的目标不一致。
- `Home.vue` 虽然信息完整，但还是典型表单页组织方式，缺少首屏节奏、视觉重点和高端产品感。
- `Result.vue` 信息很全，但目前块与块之间更像“功能列表”，而不是一个被精心导演过的沉浸式旅行产品页面。
- 结果页已存在局部 `LiquidGlass` 用法，但它尚未提升为全站统一语言，且不能直接扩散到所有区域，否则会造成性能与阅读性问题。

### 当前工程与实现风险
- `frontend/src/design-system.css` 对 `.ant-*` 覆盖较多，重构时必须以“统一 token -> 组件覆盖 -> 页面局部样式”的顺序推进，避免全局样式互相打架。
- `frontend/src/views/Result.vue` 同时承载地图、导出、编辑、图片回退与复杂视觉，任何大改都要避免破坏这些已有逻辑。
- 导出逻辑位于 `Result.vue` 内部，视觉结构重构后需要同步验证导出图片/PDF 是否仍可用。
- 仓库中还残留旧 Argon 资产和文档痕迹，但本次改造不以恢复 Argon 为目标，而是聚焦当前正在运行的 Ant Design + 自定义样式体系。

## Assumptions & Decisions
- 保留 `Ant Design Vue` 作为主交互组件基础，不新引入新的主 UI 组件体系。
- 不改后端接口、不改页面数据来源、不改生成行程、编辑、导出、地图初始化等业务逻辑。
- 允许重构页面布局与 DOM 分组，只要保留展示内容与交互结果一致。
- 全站统一苹果玻璃感，但实现手段以 CSS 玻璃层、半透明层、模糊背景、柔光高亮、圆角和空间排版为主；`LiquidGlass` 仅在最适合的局部区域保留或优化，不会无差别全站铺开。
- 动效以“克制顺滑”为准：淡入、轻微位移、hover 抬升、滚动出现，避免夸张的 3D 或强视差。
- 结果页优先任务是重做信息层次：先解决主次关系、阅读路径和节奏，再处理各个卡片细节。

## Proposed Changes

### 1. 重做全局壳层与设计令牌

#### 文件
- `frontend/src/App.vue`
- `frontend/src/design-system.css`

#### 修改内容
- 将 `App.vue` 从“普通后台式 layout”升级为更轻、更通透的应用壳层：头部改成高透明度磨砂玻璃导航条；内容区域改成更宽松的居中容器，并统一两页的顶边留白与阅读宽度；引入统一的背景层策略，如柔和渐变、光斑、模糊浮层，但保证内容对比度。
- 在 `design-system.css` 中重建设计令牌：更新主色、辅助色、中性色、边框色、阴影、模糊层、玻璃背景透明度、圆角体系、间距体系、字阶体系；将全局 Ant Design 覆盖调整为“更浅的面板、更柔的边框、更高的对比度、更大圆角”；补充统一的玻璃类、浮层类、hero 类、section 间距类、标题排版类。

### 2. 重构首页信息架构与首屏节奏

#### 文件
- `frontend/src/views/Home.vue`

#### 修改内容
- 保留表单字段与校验逻辑不变，但重做首页结构：将标题区升级为真正的 hero 区；将表单卡从单一白卡重构为更强层次的玻璃主卡 + 内部 section 容器；将“目的地与日期 / 偏好设置 / 额外要求”重新排版为更清晰的阅读节奏；统一输入框、下拉、日期选择器、checkbox 标签的高度、留白和状态反馈；提交按钮改为更具品牌感的主 CTA，并确保视觉上绝对居中；加载进度条与状态文案改成更精致的反馈模块。
- 响应式上明确三档策略：桌面、平板、手机。

### 3. 重构结果页整体层次与阅读路径

#### 文件
- `frontend/src/views/Result.vue`

#### 修改内容
- 在不改业务逻辑的前提下重新组织结果页布局，目标是更强的主次秩序：顶部操作区升级为悬浮式控制条；概览区提升为 hero summary；预算、地图、每日行程三大区域重建视觉层级；侧边导航改为更轻的玻璃导航或吸附目录；预算与概览设计成更像“产品摘要卡组”；地图区域提升为空间更大的展示模块，但保证地图初始化容器尺寸与现有逻辑兼容；每日行程面板改为更像编辑过的 travel editorial sections；景点卡继续保留大图 + 玻璃信息层方向，但统一成全站语言的一部分；编辑态和导出态按钮做更轻的状态区分。

#### 兼容要求
- 地图容器 `#amap-container` 的初始化条件不变。
- 导出图片/PDF 的 DOM 复制逻辑必须在改造后重新验证。
- 编辑模式、移动景点顺序、删除景点、图片回退逻辑都不改业务行为。

### 4. 统一玻璃语言的使用策略

#### 文件
- `frontend/src/design-system.css`
- `frontend/src/views/Home.vue`
- `frontend/src/views/Result.vue`

#### 修改内容
- 定义三类玻璃层级：主玻璃层、次玻璃层、内容层。
- 控制玻璃感出现频次：首页 hero、主表单卡、结果页 hero summary、导航、景点卡、关键控制条使用玻璃效果；长段落正文、复杂编辑区域采用更清晰的非强玻璃底。
- 统一光感、描边、阴影、模糊半径、透明度，避免“每一块玻璃都像不同组件库”。

### 5. 统一排版体系与内容密度

#### 文件
- `frontend/src/design-system.css`
- `frontend/src/views/Home.vue`
- `frontend/src/views/Result.vue`

#### 修改内容
- 建立稳定的标题层级（页面级、section 级、卡片级、说明级）。
- 统一正文、标签、注释、数值、状态文本的字号与权重。
- 对结果页高密度信息区增加更明确的留白节奏：模块间距、卡片内边距、表单控件垂直节奏、列栅格与行距。
- 强化数值信息（预算、日期、评分、时长）的视觉识别，避免和说明文本抢权重。

### 6. 响应式与动效统一

#### 文件
- `frontend/src/design-system.css`
- `frontend/src/views/Home.vue`
- `frontend/src/views/Result.vue`

#### 修改内容
- 重新检查并重写关键断点下的布局：首页 hero 缩放、表单列数调整、按钮与标签换行规则；结果页侧边导航降级策略、预算卡列数、地图高度、景点卡列数、行程内容堆叠方式。
- 加入克制顺滑的全站动效：页面进入轻淡入、卡片 hover 微抬升、section 出现轻位移、按钮 hover/按下状态更柔和。

### 7. 验证与验收

#### 文件
- `frontend/src/views/Home.vue`
- `frontend/src/views/Result.vue`
- `frontend/src/design-system.css`
- `frontend/src/App.vue`

#### 需要验证的内容
- 首页表单提交流程正常。
- 行程生成后跳转 `Result` 正常。
- 结果页地图正常加载。
- 编辑、保存、取消编辑、删除景点、移动景点顺序正常。
- 导出图片/PDF 功能仍可用。
- 首页与结果页在风格上明显属于同一产品。
- 与当前版本相比，用户第一眼能明显感知“完全不同的产品感”。
- 玻璃感统一且不过量，长内容依然清晰可读。
- `npm run build` 通过。
- 不引入新的可见报错，以及按钮不居中、层叠错乱、地图高度塌陷、导出样式失真等回归问题。

## Implementation Order
1. 先改 `App.vue` 与 `design-system.css`，建立新壳层与全局 token。
2. 再重构 `Home.vue`，先拿到首页首屏和表单卡的整体新气质。
3. 再重构 `Result.vue` 的大结构与 section 层次。
4. 最后精修 `Result.vue` 的卡片细节、编辑态与导出兼容。
5. 完成后统一做构建、交互回归与视觉检查。

## Verification Steps
- 启动前端，分别检查 `/` 与 `/result`。
- 在首页完整走一遍表单输入、提交、加载状态、跳转。
- 在结果页检查概览、预算、地图、侧边导航、每日展开、景点卡、酒店、餐饮、编辑态、导出。
- 对比重构前后截图，确认已经达到“内容相同，但视觉上像完全不同产品”的目标。
